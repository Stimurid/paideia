# UX Operator Surface Patterns

Paideia operator surfaces should expose what work is about to happen before expensive or mutating LLM actions.

Relevant surfaces:

- LLM runs;
- courses and course dynamics;
- course members;
- Kaiyona discuss;
- Kairon;
- project lab and scenario runs;
- library conflicts;
- Foundry;
- export;
- canvas fill;
- BYOK/settings and login/code flows.

Good future controls:

- prompt/source viewer;
- run profile preview;
- estimated token/cost range;
- mutation target summary;
- rollback note;
- trace link after run.

This install does not patch UI templates.

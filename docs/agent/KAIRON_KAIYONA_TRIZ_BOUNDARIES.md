# Kairon, Kaiyona, And TRIZ Boundaries

Paideia has several local thinking surfaces. They must not be collapsed into one generic “agent”.

## Kairon

Kairon is the publication / article-model layer.

Treat Kairon prompts, routes, and outputs as protected Paideia ontology unless a reviewed Kairon task explicitly requests changes.

## Kaiyona

Kaiyona is the course / project discussion and conversation layer where verified by the app.

Treat Kaiyona as a local conversational subject surface, not an Agentum runtime guard.

## Paideia TRIZ

Paideia TRIZ is pedagogical specialization:

- scenario runs;
- AgentRun;
- course/project transformation;
- L1 agents;
- Foundry and system-model surfaces.

## Quinta TRIZ

Quinta is the general TRIZ / constraint-engine lineage.

Agentum may map genealogy between Paideia and Quinta, but must not replace Paideia TRIZ with Quinta by default.

## Agentum Boundary

Agentum may provide:

- guard/control-plane docs;
- verification ladder;
- no-touch surfaces;
- prompt overlay policy;
- future eval/probe framing.

Agentum must not silently rewrite Kairon, Kaiyona, TRIZ, prompt, or scenario content.

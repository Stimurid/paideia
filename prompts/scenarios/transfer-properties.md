# Перенос свойств из выбранного кейса

Ты получаешь два объекта:
- **исходный кейс корпуса** (реальная практика),
- **проект пользователя** (замысел).

Твоя задача — описать, как перенести **архитектуру и свойства исходного кейса**
в проект пользователя, явно различая, что переносится напрямую, что — с
адаптацией, что — не переносится вовсе.

## Строгий JSON

```
{
  "summary": "<2-3 предложения, что мы переносим и зачем>",
  "transferable_directly": [
    {"property": "<свойство>", "from_case": "<как было>", "to_project": "<как станет>", "why": "<обоснование>"}
  ],
  "transferable_with_adaptation": [
    {"property": "...", "adaptation_needed": "<что нужно изменить>", "tradeoff": "<что теряем>"}
  ],
  "not_transferable": [
    {"property": "...", "blocker": "<что мешает>"}
  ],
  "new_risks_introduced": ["<риски при переносе>"],
  "open_questions": ["<что решить перед переносом>"]
}
```

Запреты: никакого «инновационно/трансформирует». Только конкретные свойства и ограничения.

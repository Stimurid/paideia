# Library Concept Extraction — концепты из фрагмента книги

Получаешь:
- book_id, title, authors
- summary книги (для контекста)
- фрагмент текста (~12 000 символов — обычно 1-2 главы)
- список уже извлечённых концептов (чтобы не дублировать)
- каталог теории Paideia: типы A-F, гипотезы H1-H5, противоречия, моды (для links)

Извлеки **3-8 концептов** из этого фрагмента.

## Что такое концепт

- Авторский термин / понятие с собственным значением (например «зона ближайшего развития» у Выготского)
- Эмпирическое обобщение (например «студент с AI учится медленнее в первый месяц»)
- Методический приём / паттерн (например «фейдинг поддержки»)
- Гипотеза автора (например «миметическое желание формируется в контакте»)

**Не концепт:** общеизвестный термин без авторского значения, бытовая фраза, перечень.

## Возврат: строгий JSON

```json
{
  "concepts": [
    {
      "name": "<краткое название концепта, не более 60 символов>",
      "definition": "<1-3 предложения определения/раскрытия>",
      "quote": "<ключевая цитата если есть, до 300 символов>",
      "page_hint": <int номер страницы или null>,
      "importance": <1-5>,
      "links": [
        {"kind": "type|hypothesis|tension|mode|case", "id": "<id из каталога>", "relation": "supports|illustrates|contradicts|extends"}
      ]
    }
  ],
  "skipped": ["<упомянутые но не извлечённые термины, с пояснением>"]
}
```

## Правила
- Не более 8 концептов из одного фрагмента.
- Не дублируй концепты из existing_concepts.
- Каждый концепт **должен** иметь definition. Quote — желательно.
- links — только если есть реальная связь, не натягивай.
- importance: 5 = ядро книги, 1 = маргинальный.
- Не выдумывай номера страниц — null если не уверен.

# Bulk extractor — пройтись по сегменту длинного документа и вытащить кейсы

Получаешь один сегмент длинного источника (раздел отчёта, слайд, страница PDF).
Решаешь: есть ли в нём один или несколько кейсов внедрения ИИ в образовании?

## Что считать кейсом

Реальная или прототипная практика, в которой:
- есть организация (университет, школа, корпорация, министерство, лаб),
- есть встраивание LLM/GenAI/агентов как функционально различённого участника,
- либо есть архитектура (роли, оркестрация, контур governance) — пусть даже на
  уровне презентации.

Что **не** кейс: общие рассуждения, цитаты учебников, абстрактные тезисы про
«ИИ изменит образование», списки литературы, заголовки без описания.

## Возврат: строгий JSON

```
{
  "candidates": [
    {
      "id": "<kebab-slug>",
      "name": "<Organization> · <product/initiative>",
      "organization": {"name": "...", "type": "U|R|C|S|N", "country": "ISO-2 или INT"},
      "ai": {"pattern": "A..F|null", "agentivity": 0-6 или null, "technologies": ["..."]},
      "lifecycle_stage": "idea|poc|pilot|rollout|rolled-back",
      "summary": "<2-4 предложения о сути кейса>",
      "confidence": "low|medium|high",
      "source_excerpt": "<≤500 символов оригинального текста, по которому ты сделал вывод>"
    }
  ],
  "is_relevant": true|false,
  "note": "<если is_relevant=false — короткое объяснение почему>"
}
```

Если в сегменте нет кейсов — верни `{"candidates": [], "is_relevant": false, "note": "..."}`.

## Запреты

- Не выдумывай. Если организация не названа — id ставь `unknown-<slug>` и
  confidence=low.
- Никаких упоминаний "я", "as an AI" в summary/note.
- Не повторяй один и тот же кейс несколько раз в одном сегменте.
- Не предлагай в один сегмент более 3 кандидатов (если больше — это значит, что
  сегмент был нарезан слишком крупно).

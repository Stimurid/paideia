# Match explainer — объяснить пару (ситуация ↔ аналог)

Получаешь: описание ситуации пользователя (его параметры) и одного
конкретного кейса-аналога из корпуса. Задача: коротко (3-5 предложений)
объяснить, **почему этот кейс похож** и **в чём ключевые отличия,
которые ограничивают перенос**.

## Что важно

- Не цитировать общие принципы — указывать **конкретные совпадения**
  (агентность, паттерн, ролевая модель, transformation_mode).
- Не сглаживать различия. Если кейс corporate, а пользователь — региональный
  RU-вуз, явно сказать «учти, что governance-контекст другой».
- Не выдумывать данные о кейсе, которых нет в карточке.

## Возврат: строгий JSON

```
{
  "case_id": "<id>",
  "why_similar": "<2-3 предложения с конкретными совпадениями>",
  "key_differences": "<2-3 предложения с конкретными отличиями>",
  "transfer_advice": "<1-2 предложения: что нужно адаптировать>"
}
```

# Semester Simulator — прогон программы по неделям

Получаешь SystemModel и параметры (длина в неделях по умолчанию = 16,
число студентов, формат).

Симулируй неделя-за-неделей что произойдёт. Учитывай:
- мотивация падает к неделе 5-8 (типичная U-curve)
- технические сбои случаются (API недоступен 1-2 недели в семестре)
- 1-2 студента отвалятся (drop-out)
- к экзамену накапливается отставание
- группа делится на 3 кластера успешности

Для каждой недели:
- что планируется по программе
- что реально произойдёт (с учётом рисков)
- какие сигналы получит преподаватель
- какое действие нужно предпринять

## Возврат: строгий JSON

```json
{
  "weeks": [
    {
      "week": 1-16,
      "phase": "onboarding | building | peak | crisis | recovery | closing",
      "planned": "<что по программе>",
      "actual_likely": "<что реально произойдёт>",
      "signals_to_teacher": ["..."],
      "intervention_needed": "<что делать или null>",
      "risk_level": 1-5
    }
  ],
  "predicted_dropout": "<число>/<group_size>",
  "weakest_weeks": [<numbers>],
  "strongest_weeks": [<numbers>],
  "overall_design_score": 1-5
}
```

Запреты:
- Не «всё пойдёт хорошо» — реализм с типичными провалами
- Конкретные сигналы (а не «студенты будут менее вовлечены»)
- Не более 16 недель

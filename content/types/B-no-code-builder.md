## Описание

Тип B — платформа, позволяющая преподавателю или подразделению собирать
собственного AI-бота под курс или административный процесс без программирования:
загрузка материалов, описание правил поведения, развёртывание в управляемой
среде. Сюда же относятся LMS-встроенные микророли (summaries, rubric generation,
grading assistance) — это та же логика конструктора, только унифицированная на
уровне LMS.

В отличие от типа A (governed access), B требует, чтобы преподаватель явно
проектировал AI-роль. В отличие от типа C (pedagogy-shaped dialogue), B не
накладывает педагогической формы на сам диалог — это инструментальный слой.

## Условия принадлежности

Кейс инстанциирует тип B, если выполняется минимум один из критериев:

- есть отдельный конструктор для создания локальных ботов (MyAI Builder, Maizey,
  ZotGPT Creator, UT Sage, IgniteAI Agent);
- AI-функции встроены в LMS как набор операционных микроролей (summaries,
  rubric gen, AI Design Assistant);
- появляется per-course / per-department AI-объект с собственным контекстом.

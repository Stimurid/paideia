## Что ещё хотели, но не сделали

### A. Полноценные ТРИЗ-движки (главное)

Из изначального плана: «**форк твоего инженерного ТРИЗ-движка под педдизайн**. Старый промпт из «промпта для элективного» — частичный донор: многоосевая модель пространства, типология траекторий, сценография диалога сохраняются; метаязык переписывается на ТРИЗ-нотацию (противоречие, ИКР, веполь, 40 приёмов под образовательные ресурсы)».

Не сделано:
- **40 ТРИЗ-приёмов в педагогической интерпретации** как структурированный yaml-каталог (`taxonomy/triz_methods_edu.yaml`)
- **Веполь-анализ** с визуальной триадой «студент — содержание — посредник» (сейчас просто JSON-промпт)
- **Итеративный ИКР-флоу** (диалог: цель → ИКР → препятствия → переформулировка)
- **8 линий эволюции технических систем**, адаптированные под образование, как справочник + диагностика
- **Противоречия как граф** — пары требований сохраняются в БД как связи, видны на странице проекта

### B. Стейкхолдер-обстрел как связный workflow

Сейчас 4 отдельных промпта (карта, атака, трение, нормативка). Должны быть **связанные шаги**:
- Сущность `stakeholders` в БД с типами/интересами/рычагами
- Авто-генерация карты по канвасу проекта
- Атака от лица каждого → синтез слабых мест проекта

### C. Симулятор и стресс-тесты — реальный UI

- Симулятор первого семестра как **таймлайн с неделями** (не JSON)
- What-if матрица как таблица «сценарий × секция канваса × выдерживает»
- Anti-agent-washing как **скоринг** (заявленная агентность vs фактическая, с расшифровкой по критериям)

### D. Дикие идеи с настоящей реализацией

- **Time-machine** — на основе эволюционных трендов (опирается на A.4)
- **Genealogy** — граф «родители/дети» кейса в БД, визуальный
- **Counter-corpus** — отдельная выборка кейсов-провалов с явным юзкейсом «обстреляй мой проект всем что провалилось»
- **Hypothesis forge** — UI создания H6 с накоплением свидетельств

### E. Метакогнитив с реальным UI

- **Slop-минёр** должен **подсвечивать места в тексте** канваса (инлайн-аннотации), не просто отдавать JSON
- **Ассумпции** — список с возможностью добавить контр-факт
- **Blind-spots** с прямой привязкой к пустым секциям канваса

### F. Из «прежнего канваса» — настоящий инструмент проектирования

- **Типология преподавателей** (готовый замысел / растерянность / нужны идеи) — проектный визард задаёт разные вопросы
- **Ролевая модель ТюмГУ** как обязательный раздел канваса проекта: картограф / организатор / медиатор / хранитель мотивации + ИИ-персоны (сейчас просто текстовые поля)
- **Паспорт AI-персоны** как самостоятельная сущность проекта (роль, функции, ограничения, точки интеграции)
- **Проектный визард** — поэтапное заполнение через вопросы вместо пустой формы
- **Сценография диалога** (узел понимания → воплощения → резонанса → прояснения) как UX-паттерн визарда

### G. Нормативка и документы РФ

- Привязка РПД к **ФГОС** (с подбором кодов компетенций)
- Шаблон заявки в **Минобрнауки** на основе реальной структуры
- Шаблон бизнес-плана с **CAPEX/OPEX/CAC/LTV**
- Силлабус по образцам ТюмГУ-кейсов (берём из корпуса)

### H. Доработки инфраструктуры

- **UI просмотра git-истории** секции канваса (откат правки)
- **Кеши** всех тяжёлых LLM-вызовов (сейчас только pptx-extract кешируется)
- **LLM-аудит с токенами и стоимостью** по моделям (сейчас только время и статус)
- **Multi-tenant** — basic_auth на каждого пользователя или OAuth (опц.)
- **Версии моделей** в аудите (для воспроизводимости)

### I. Глубокая Wave-runner

Сейчас один промпт. Должен быть **7-фазный flow** как в оригинальных wave-1..wave-4:
- расширение терминологии → прочёс → проверка покрытия → диагностика → синтез → фальсификация → верификация
- сравнение волн как явный дифф с визуальной разметкой сдвигов

### J. Связи и шаринг

- Шеринг результатов сценариев (короткий URL, экспорт в md)
- Авто-применение результатов в канвас (slop-минёр нашёл мины → кнопка «применить правки»)
- Связки между сценариями (стейкхолдер ↔ секция проекта; противоречие ↔ ИКР)

### K. Мелочи и боли

- Многоосевая модель пространства из «электривного промпта»: деятельности × инструменты × оси × уровни входа — как параллельная онтология рядом с фасетами
- Visualization карты с **цветами по семействам**, легендой, фильтрами по агентности
- 2D scatter сейчас примитивный — стоит улучшить (зум, фильтры, click-to-detail)
- Telegram-бот для быстрых запросов (опц.)
- Контр-сигналы как первоклассная сущность с UI «добавить новый контр-сигнал»
